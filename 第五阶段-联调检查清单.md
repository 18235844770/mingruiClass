# 第五阶段联调检查清单（收尾版）

> 目标：在进入第六阶段前，确认管理员模块可稳定闭环。  
> 适用角色：`admin`（主）、`owner`（日志查询回归）。

---

## 一、环境准备

- [ ] 后端已启动，数据库连接正常
- [ ] 前端已启动，可正常登录
- [ ] 已有基础数据：至少 1 个管理员、1 个销售、2 个校区
- [ ] 浏览器本地缓存中无脏 token（必要时退出重登）

---

## 二、用户管理 `/users`

### 2.1 列表与筛选
- [ ] 默认列表可加载，展示用户名/角色/状态/校区
- [ ] 用户名筛选生效
- [ ] 角色筛选生效
- [ ] 状态筛选生效
- [ ] 校区筛选生效
- [ ] 分页与每页条数切换生效

### 2.2 新建与编辑
- [ ] 可新建用户（用户名唯一、密码>=6、可绑定多个校区）
- [ ] 新建后列表可见，详情数据正确
- [ ] 可编辑角色/状态/校区绑定
- [ ] 不填重置密码时原密码不变
- [ ] 填写重置密码后可用新密码登录

### 2.3 删除规则
- [ ] 删除普通用户成功
- [ ] 删除当前登录用户被拒绝并返回可读提示
- [ ] 删除最后一个管理员被拒绝并返回可读提示

---

## 三、角色管理 `/roles`

### 3.1 列表与筛选
- [ ] 角色列表加载正常
- [ ] 编码筛选生效
- [ ] 名称筛选生效
- [ ] 分页生效

### 3.2 新建与编辑
- [ ] 新建角色成功（编码唯一）
- [ ] 重复编码创建被拒绝并返回可读提示
- [ ] 编辑角色名称成功

### 3.3 删除规则
- [ ] 被用户引用的角色删除被拒绝
- [ ] 未被引用的角色可删除

---

## 四、校区管理 `/campuses`

### 4.1 列表与筛选
- [ ] 校区列表加载正常
- [ ] 名称筛选生效
- [ ] 状态筛选生效
- [ ] 分页生效

### 4.2 新建与编辑
- [ ] 新建校区成功
- [ ] 编辑名称/状态成功

### 4.3 删除规则
- [ ] 被 `students` 或 `user_campuses` 关联的校区删除被拒绝
- [ ] 无关联校区可删除

---

## 五、日志管理 `/logs`

### 5.1 列表与筛选
- [ ] 日志列表加载正常
- [ ] 按操作人筛选生效
- [ ] 按模块筛选生效
- [ ] 按校区筛选生效
- [ ] 按时间范围筛选生效
- [ ] 分页生效

### 5.2 详情与删除
- [ ] 日志详情抽屉可打开，`detail` 可读
- [ ] 删除日志后列表不可见（软删）
- [ ] 删除失败时展示后端错误信息

### 5.3 写操作追溯
- [ ] 用户新增/编辑/删除后生成 `users` 模块日志
- [ ] 角色新增/编辑/删除后生成 `roles` 模块日志
- [ ] 校区新增/编辑/删除后生成 `campuses` 模块日志

---

## 六、权限回归

- [ ] `admin` 可访问 `/users` `/roles` `/campuses` `/logs`
- [ ] `owner` 可访问 `/logs`，不可访问管理员配置页
- [ ] `sales` 不可访问管理员配置页
- [ ] `GET /api/campuses/mine` 仍可正常用于销售选校区

---

## 七、收尾判定

- [ ] 上述关键项全部通过
- [ ] 发现问题已记录并修复
- [ ] 第五阶段可判定完成，进入第六阶段开发
