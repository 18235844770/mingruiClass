# 第九阶段：老板仪表盘与报表 — 任务拆分（执行版）

> 依据：`阶段任务清单.md` 第九阶段、`产品设计文档V2.0.md`。  
> **前置**：第六～第八阶段已完成（学生、课程、消课、提醒链路可用且数据完整）。

---

## 阶段目标（不变）

实现老板视角的数据洞察能力：仪表盘核心指标、报表筛选分析、导出交付，并保证统计口径可复核。

---

## 9.1 仪表盘核心指标

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 9.1.1 | 在读人数 | 统计有效学生数（按未软删口径） |
| 9.1.2 | 剩余课时 | 统计课程剩余课时总量（含负值） |
| 9.1.3 | 总课时 | 统计课程总课时总量 |
| 9.1.4 | 总收入 | 统计已录入总金额汇总（按约定口径） |
| 9.1.5 | 未收入课时费用 | 基于策略计算“未收入课时费用” |

---

## 9.2 筛选维度（仪表盘）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 9.2.1 | 校区筛选 | 支持按单校区/全部校区筛选 |
| 9.2.2 | 时间范围筛选 | 支持开始/结束时间范围 |
| 9.2.3 | 口径一致性 | 同一筛选条件下仪表盘与报表数据一致 |
| 9.2.4 | 默认值策略 | 未选择筛选时有明确默认口径（如全量） |

---

## 9.3 未收入课时费用策略

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 9.3.1 | 策略封装 | 在 service 层封装计算规则（不可散落在 controller） |
| 9.3.2 | 参数可配置 | 支持按配置项调整（如单价来源、舍入规则） |
| 9.3.3 | 可解释性 | 返回结果可追溯到输入参数与公式 |
| 9.3.4 | 边界处理 | 负课时、空值、零值场景处理明确 |

---

## 9.4 报表查询能力

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 9.4.1 | 筛选项 | 校区、销售、课程类型、日期范围 |
| 9.4.2 | 数据明细 | 支持按学生/课程/消课维度聚合或明细展示 |
| 9.4.3 | 分页与排序 | 支持分页；关键列支持排序 |
| 9.4.4 | 与仪表盘联动 | 可从仪表盘入口跳转报表并带筛选条件 |

---

## 9.5 导出能力（Excel/PDF）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 9.5.1 | 导出接口 | `GET /api/reports/export`（建议异步或流式） |
| 9.5.2 | Excel 导出 | 字段齐全、中文表头、时间格式统一 |
| 9.5.3 | PDF 导出 | 布局可读、分页合理、关键汇总可见 |
| 9.5.4 | 筛选继承 | 导出结果与当前筛选条件一致 |

---

## 9.6 权限策略

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 9.6.1 | 老板只读 | `owner` 仅查看/导出，不可写 |
| 9.6.2 | 管理员可查看 | `admin` 可访问报表与仪表盘（按产品） |
| 9.6.3 | 销售隔离 | `sales` 默认不可访问老板视角页 |
| 9.6.4 | 接口守卫 | 路由与接口双重校验权限 |

---

## 9.7 横切要求（第九阶段）

| 序号 | 任务 | 说明 |
| --- | --- | --- |
| 9.7.1 | 统计逻辑集中 | 全部统计逻辑在 service 层封装 |
| 9.7.2 | 时间格式统一 | 前端展示 `yyyy-mm-dd hh:mm` |
| 9.7.3 | 错误信息可读 | 参数错误、权限错误、导出失败返回可读 message |
| 9.7.4 | 性能约束 | 大查询支持索引、分页、必要缓存 |
| 9.7.5 | 结果可复核 | 提供口径说明，支持 SQL/样本复核 |

---

## 9.8 后端实现建议（Nest）

| 模块 | 任务建议 |
| --- | --- |
| `DashboardModule` | 核心指标接口、筛选参数处理 |
| `ReportsModule` | 报表查询、导出、汇总与明细能力 |
| `Students/Consumptions` | 复用底层查询能力，避免重复 SQL |

---

## 9.9 前端页面建议（老板/管理员）

| 路由 | 页面 | 备注 |
| --- | --- | --- |
| `/dashboard` | 老板仪表盘 | 指标卡 + 趋势/分布 + 筛选区 |
| `/reports` | 报表管理 | 筛选表单 + 表格 + 导出按钮 |

> 图表建议：先用稳定图表库基础组件，优先确保数据口径正确。

---

## 9.10 验收标准（与总清单对齐）

- [ ] 仪表盘核心指标完整且口径一致
- [ ] 支持按校区/时间范围筛选，结果准确
- [ ] 报表支持多维筛选并可导出 Excel/PDF
- [ ] 老板只读、管理员可查看权限生效
- [ ] 统计结果可复核，关键数字一致

---

## 9.11 与第十阶段边界

- 第九阶段完成“功能可用 + 口径稳定”。  
- 全链路压测、回归、缺陷清零和验收报告由第十阶段完成。

## 建议执行顺序

1. 先固化统计口径与 service 封装  
2. 实现仪表盘核心指标接口  
3. 实现报表查询与筛选  
4. 接入导出能力（先 Excel，后 PDF）  
5. 前端仪表盘/报表页面联调  
6. 权限与口径复核，完成 9.10 自检
