# 第六阶段联调检查清单（学生与课程）

> 目标：对齐 `第六阶段-任务拆分.md` 的 6.9 验收项，完成学生与课程管理链路前后端联调自检。

---

## 0. 联调前准备

- [ ] 后端已启动（`/api` 前缀可访问）
- [ ] 前端已启动并可使用销售账号登录
- [ ] 销售账号已完成校区选择
- [ ] 已准备校验数据：至少 1 个同校区重名样本、1 个跨校区样本

---

## 1. 学生列表与筛选（`/students`）

### 用例 1：列表加载与分页

预期：

- [ ] 学生列表可正常加载
- [ ] 分页切换生效
- [ ] 每页条数切换生效

### 用例 2：筛选条件

分别测试：姓名、手机号、交费状态、低课时、负课时。

预期：

- [ ] 单条件筛选生效
- [ ] 多条件组合筛选生效
- [ ] 重置后回到默认列表

### 用例 3：标签展示

预期：

- [ ] 低课时显示“低课时”标签
- [ ] 负课时显示“负课时”标签并高亮
- [ ] 未交费显示“欠费”标签

---

## 2. 学生创建与重名校验

### 用例 4：正常创建

1. 新建学生，填写姓名、总金额、交费状态（手机号与备注可选）
2. 提交保存

预期：

- [ ] 创建成功，列表中可见
- [ ] `campusId` 强制为当前校区
- [ ] `createdBy` 为当前销售

### 用例 5：同校区重名

1. 在同一校区创建同名学生

预期：

- [ ] 被拒绝并返回可读错误（如“同一校区下学生姓名不能重复”）
- [ ] 不会落库

### 用例 6：跨校区同名

预期：

- [ ] 不同校区同名可创建（不冲突）

---

## 3. 课程多条录入

### 用例 7：一次录入多条课程

1. 在学生列表点击“录入课程”
2. 一次填写多条课程：课程名、价格、总课时、剩余课时、类型、备注
3. 提交

预期：

- [ ] 多条课程一次保存成功
- [ ] 课程字段完整可回显
- [ ] 学生列表剩余课时为课程剩余课时聚合值

### 用例 8：规则边界

预期：

- [ ] 不校验“课程价格总和 == 学生总金额”
- [ ] 仅做字段完整性与数值合法性校验

---

## 4. 学生编辑与删除

### 用例 9：编辑学生

预期：

- [ ] 可编辑姓名、手机号、总金额、交费状态、备注
- [ ] 编辑后列表与详情数据同步更新

### 用例 10：删除学生

1. 点击删除并确认

预期：

- [ ] 二次确认弹窗出现
- [ ] 删除后列表不可见
- [ ] 软删策略生效（`students.deleted_at` 赋值）
- [ ] 关联课程按策略处理（已软删或不可见）

---

## 5. 学生详情聚合

### 用例 11：详情信息完整性

1. 打开学生详情

预期：

- [ ] 展示学生基础信息
- [ ] 展示课程列表（价格/总课时/剩余课时/类型/备注）
- [ ] 展示消课历史摘要（记录数、总消课课时）

---

## 6. 权限与数据范围

### 用例 12：销售数据范围

预期：

- [ ] 销售仅可查看当前校区 + 自己创建的学生
- [ ] 销售不可编辑/删除他人创建学生
- [ ] 通过 URL 直接访问他人学生详情会被拒绝

### 用例 13：管理角色回归（如已开放）

预期：

- [ ] 管理员可按产品规则查看更广数据范围
- [ ] 非授权角色不会暴露销售页面操作能力

---

## 7. 操作日志与可追溯

### 用例 14：学生写操作日志

预期：

- [ ] 学生新增/编辑/删除写入 `operation_logs`
- [ ] 日志含操作人、模块、目标 ID、关键字段变更

### 用例 15：课程写操作日志

预期：

- [ ] 课程新增写入 `operation_logs`
- [ ] 模块与详情字段可用于追溯

---

## 8. 第六阶段 6.9 验收打勾

- [ ] 学生列表支持姓名/手机号/交费状态/低课时/负课时筛选
- [ ] 学生可新增、编辑、删除；姓名同校区唯一生效
- [ ] 课程支持一次录入多条，字段完整可保存
- [ ] 学生详情可查看基础信息、课程和历史消课摘要
- [ ] 销售仅能在当前校区 + 自己数据范围内操作
