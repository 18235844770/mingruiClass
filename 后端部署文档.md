# 明睿教务系统后端部署文档

本文档仅覆盖后端服务（NestJS + Prisma）部署流程，适用于 Ubuntu 22.04 生产环境。

## 1. 部署目标

- 后端服务目录：`/home/mingrui/backend`
- 运行端口：`3000`
- 进程守护：`pm2`
- 数据库：`PostgreSQL`

## 2. 前置依赖

请先确保服务器已安装：

- Git
- Node.js 20.x 与 npm
- PM2
- PostgreSQL

如未安装，可先执行 `服务器依赖安装文档.md`。

## 3. 获取代码

```bash
cd /home/mingrui

# 替换为你的仓库地址
git clone <your-repo-url> backend
```

## 4. 配置环境变量

```bash
cd /home/mingrui/backend
cp .env.example .env
```

编辑 `backend/.env`（至少确认以下字段）：

```env
DATABASE_URL="postgresql://mingrui_user:StrongPassword_ChangeMe@127.0.0.1:5432/mingrui_class?schema=public"
JWT_SECRET="replace-with-a-very-strong-random-secret"
JWT_EXPIRES_IN="1d"
CORS_ORIGINS="https://your-domain.com"
PORT=3000
```

## 5. 初始化数据库

推荐使用仓库脚本初始化数据库账号与库：

```bash
sudo -u postgres psql -f /home/mingrui/backend/scripts/init-db.sql
```

然后执行 Prisma 迁移与种子：

```bash
cd /home/mingrui/backend
npm install
npx prisma generate
npx prisma migrate deploy
npm run prisma:seed
```

> 若种子写入默认账号，请上线后立即修改默认密码。

## 6. 构建并启动后端

```bash
cd /home/mingrui/backend
npm run build
pm2 start dist/src/main.js --name mingrui-backend
pm2 save
pm2 startup
```

查看运行状态与日志：

```bash
pm2 status
pm2 logs mingrui-backend
```

## 7. 健康检查

服务启动后执行：

```bash
curl http://127.0.0.1:3000/api/health
```

期望返回（示例）：

```json
{"ok":true,"service":"mingrui-backend"}
```

## 8. 更新发布流程

```bash
cd /home/mingrui/backend
git pull
npm install
npm run build
npx prisma migrate deploy
pm2 restart mingrui-backend
```

## 9. 常见问题排查

- 服务启动失败：执行 `pm2 logs mingrui-backend` 查看堆栈
- 数据库连接失败：检查 `DATABASE_URL`、PostgreSQL 服务状态、端口与防火墙
- Prisma 迁移失败：执行 `npx prisma migrate status` 检查迁移状态
- 接口 404：确认全局前缀为 `/api`，并检查访问路径
