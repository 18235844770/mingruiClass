# 第八阶段：低课时提醒与销售首页 — 任务拆分（执行版）

> 依据：`阶段任务清单.md` 第八阶段、`产品设计文档V2.0.md`。  
> **前置**：第七阶段已完成（消课记录、剩余课时扣减、负课时可识别）。

---

## 阶段目标（不变）

强化销售端提醒能力：销售首页固定展示低课时学生，并支持筛选、跳转与跟进，确保提醒数据实时且严格受校区与创建人范围约束。

---

## 8.1 低课时数据接口

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 8.1.1 | 低课时查询接口 | `GET /api/students/low-hours`（或在 `students` 列表加 `lowHoursOnly=true`） |
| 8.1.2 | 默认阈值 | 默认返回 `remainingHours < 3` 的学生 |
| 8.1.3 | 可配置阈值（可选） | 支持 query 传阈值（如 `threshold=3`），未传走默认 |
| 8.1.4 | 返回字段 | 学生基础信息 + 课程维度剩余课时 + 欠费状态 + 负课时标识 |

---

## 8.2 销售首页提醒区

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 8.2.1 | 首页提醒组件 | 销售首页固定区域展示低课时学生列表 |
| 8.2.2 | 状态标签 | 同时标记 `低课时`、`负课时`、`欠费` |
| 8.2.3 | 排序策略 | 默认按“剩余课时升序 + 最近更新时间降序” |
| 8.2.4 | 空态处理 | 无低课时数据时显示友好空态文案 |

---

## 8.3 低课时独立列表页

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 8.3.1 | 路由页面 | `/low-hours` 独立页面可访问 |
| 8.3.2 | 筛选条件 | 支持姓名/手机号/欠费状态/负课时等筛选 |
| 8.3.3 | 分页支持 | 统一 `{ data, total, page, pageSize }` |
| 8.3.4 | 与学生页一致性 | 列字段与标签口径与 `students` 保持一致 |

---

## 8.4 业务跳转联动

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 8.4.1 | 跳转学生详情 | 从提醒列表一键进入学生详情 |
| 8.4.2 | 跳转消课页 | 从提醒列表一键进入消课页面并预选学生/课程（可选） |
| 8.4.3 | 返回体验 | 详情/消课返回后保留筛选与分页状态 |

---

## 8.5 数据范围与权限

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 8.5.1 | 销售可见范围 | 仅当前校区 + 当前销售创建范围（沿用第四阶段） |
| 8.5.2 | 防越权校验 | 禁止通过 query 伪造跨校区数据 |
| 8.5.3 | 非销售角色策略 | 管理员/老板访问行为按产品定义（可先不开放） |

---

## 8.6 横切要求（第八阶段）

| 序号 | 任务 | 说明 |
| --- | --- | --- |
| 8.6.1 | 时间格式统一 | 前端时间展示统一 `yyyy-mm-dd hh:mm` |
| 8.6.2 | 错误提示可读 | 接口异常返回中文可读 message |
| 8.6.3 | 性能可接受 | 首页提醒接口首屏响应稳定（建议分页或限制条数） |
| 8.6.4 | 与第七阶段一致 | 低课时与负课时判定口径与消课后状态一致 |
