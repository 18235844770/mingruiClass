# 明睿教务系统数据库初始化文档

本文档用于在服务器上完成 PostgreSQL 数据库初始化，并与后端（NestJS + Prisma）打通。

适用环境：

- 操作系统：Ubuntu 22.04
- 数据库：PostgreSQL 14+
- 项目路径：`/home/mingrui/backend`

## 1. 前置条件

请先确认以下条件已满足：

- 已安装并启动 PostgreSQL
- 已拉取项目代码到服务器
- 已安装 Node.js 与 npm

快速检查：

```bash
psql --version
sudo systemctl is-active postgresql
node -v
npm -v
```

## 2. 创建数据库用户与数据库

可直接执行仓库脚本（推荐）：

```bash
sudo -u postgres psql -f /home/mingrui/backend/scripts/init-db.sql
```

或手工进入 PostgreSQL 控制台：

进入 PostgreSQL 控制台：

```bash
sudo -u postgres psql
```

执行以下 SQL（请替换强密码）：

```sql
CREATE USER mingrui_user WITH PASSWORD 'StrongPassword_ChangeMe';
CREATE DATABASE mingrui_class OWNER mingrui_user;
GRANT ALL PRIVILEGES ON DATABASE mingrui_class TO mingrui_user;
\q
```

说明：

- `mingrui_user`：业务数据库账号
- `mingrui_class`：业务数据库名
- 生产环境请使用高强度随机密码

## 3. 配置后端数据库连接

进入后端目录并准备环境变量文件：

```bash
cd /home/mingrui/backend
cp .env.example .env
```

编辑 `.env`，至少确保以下变量正确：

```env
DATABASE_URL="postgresql://mingrui_user:StrongPassword_ChangeMe@127.0.0.1:5432/mingrui_class?schema=public"
JWT_SECRET="replace-with-a-very-strong-random-secret"
JWT_EXPIRES_IN="1d"
PORT=3000
```

注意：

- `DATABASE_URL` 中用户名、密码、数据库名必须与第 2 步一致
- 如果数据库不在本机，替换 `127.0.0.1` 为实际数据库地址

## 4. 安装依赖并生成 Prisma 客户端

```bash
cd /home/mingrui/backend
npm install
npx prisma generate
```

## 5. 执行数据库迁移

生产环境推荐使用：

```bash
cd /home/mingrui/backend
npx prisma migrate deploy
```

说明：

- `migrate deploy` 会按已有迁移文件顺序执行，不会创建新迁移
- 适合测试/生产环境标准发布流程

## 6. 初始化种子数据

```bash
cd /home/mingrui/backend
npm run prisma:seed
```

如仓库默认种子存在演示账号，请上线后立即修改默认密码。

## 7. 初始化结果验收

### 7.1 检查迁移状态

```bash
cd /home/mingrui/backend
npx prisma migrate status
```

期望：迁移状态为已应用（up to date）。

### 7.2 检查数据库是否有业务表

```bash
sudo -u postgres psql -d mingrui_class -c "\dt"
```

期望：能看到 Prisma 迁移表及业务表。

### 7.3 检查后端连通性（启动后）

```bash
curl http://127.0.0.1:3000/api/health
```

期望返回（示例）：

```json
{"ok":true,"service":"mingrui-backend"}
```

## 8. 常见问题排查

- `P1000`（认证失败）：检查 `DATABASE_URL` 的用户名和密码
- `P1001`（无法连接数据库）：检查数据库地址、端口、服务状态和防火墙
- `permission denied`：确认数据库所有者和授权是否正确
- 迁移失败：先执行 `npx prisma migrate status` 定位失败迁移，再按错误信息修复

## 9. 安全建议

- 禁止在文档和仓库中明文提交生产数据库密码
- 数据库账号最小权限原则：按业务分配权限，不共用超级管理员账号
- 定期备份数据库，并演练恢复流程
- 若使用公网数据库，务必配置白名单 IP
