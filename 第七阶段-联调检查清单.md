# 第七阶段联调检查清单（消课管理）

> 目标：对齐 `第七阶段-任务拆分.md` 的 7.9 验收项，完成前后端联调自检。

---

## 0. 联调前准备

- [ ] 后端已启动（`/api` 前缀可访问）
- [ ] 前端已启动并可使用销售账号登录
- [ ] 销售账号已绑定至少一个校区
- [ ] 至少有 1 名学生，且名下有至少 1 条课程

---

## 1. 消课录入与扣减

### 用例 1：正常消课

1. 进入 `消课管理` 页面
2. 选择学生与课程，填写：
   - `consumedHours = 1`
   - `consumptionTime = 当前时间`
3. 点击“确认消课”

预期：

- [ ] 提交成功，提示“消课成功”
- [ ] 数据库新增 1 条 `course_consumptions` 记录
- [ ] 对应课程 `student_courses.remaining_hours` 被扣减 1

### 用例 2：超额消课（允许负课时）

1. 选择某课程，输入大于剩余课时的消课值（例如剩余 0.5，消课 2）
2. 提交

预期：

- [ ] 提交成功，不被阻断
- [ ] `remaining_hours` 允许为负值
- [ ] 列表出现“负课时”标签/高亮

### 用例 3：非法参数校验

依次测试：

- `consumedHours = 0`
- `consumedHours < 0`
- `consumptionTime` 为空

预期：

- [ ] 前端拦截或后端返回错误提示
- [ ] 不会生成消课记录

---

## 2. 低课时/负课时状态

### 用例 4：低课时阈值命中

1. 将课程剩余课时消到 `< 3` 且 `>= 0`
2. 查看消课记录列表与学生列表

预期：

- [ ] 消课记录返回 `isLowHours = true`
- [ ] 未负数时不显示负课时标签
- [ ] 学生列表可看到低课时状态（若已实现相关展示）

### 用例 5：负课时命中

1. 继续消课使剩余课时 `< 0`

预期：

- [ ] 消课记录返回 `isNegativeHours = true`
- [ ] 前端有负课时高亮/标签

---

## 3. 消课记录查询

### 用例 6：按学生筛选

预期：

- [ ] 仅返回该学生相关消课记录

### 用例 7：按课程筛选

预期：

- [ ] 仅返回该课程相关消课记录

### 用例 8：按日期范围筛选

预期：

- [ ] 仅返回范围内记录

---

## 4. 重复消课提醒（仅提醒）

### 用例 10：命中疑似重复

步骤：

1. 同一课程，先提交一条消课（例如 1 课时，当前时间）
2. 在临近时间内再次提交相同课时

预期：

- [ ] 首次提交第二条时出现“疑似重复”确认弹窗
- [ ] 取消后不落库
- [ ] 点击“继续提交”后落库成功
- [ ] 业务流程不被强阻断

---

## 5. 操作日志可追溯

### 用例 11：检查操作日志

在“日志管理”中筛选模块 `consumptions`，打开详情。

预期：

- [ ] 每次消课都有对应 `operation_logs` 记录
- [ ] `detail` 至少包含：
  - 学生/课程信息
  - 消课课时与消课时间
  - 扣减前后剩余课时
  - `duplicateWarningAcknowledged`
- [ ] 可通过日志中的 `consumptionId` 回溯到 `course_consumptions.id`

---
