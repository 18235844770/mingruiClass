# 第五阶段接口摘要（管理员模块）

> 基础前缀：`/api`  
> 鉴权：`Authorization: Bearer <token>`  
> 管理接口默认角色：`admin`（日志查询支持 `admin` / `owner`）

---

## 1) 校区管理 `campuses`

### `GET /campuses`
- 说明：校区列表（分页 + 筛选）
- Query：
  - `page` number（默认 1）
  - `pageSize` number（默认 10，最大 100）
  - `name` string（可选，模糊）
  - `status` string（可选）
- 响应：
```json
{
  "data": [],
  "total": 0,
  "page": 1,
  "pageSize": 10
}
```

### `POST /campuses`
- 说明：新建校区
- Body：
```json
{
  "name": "浦东校区",
  "status": "active"
}
```

### `PATCH /campuses/:id`
- 说明：编辑校区
- Body（可选字段）：
```json
{
  "name": "浦东新区校区",
  "status": "inactive"
}
```

### `DELETE /campuses/:id`
- 说明：删除校区（若被学生或用户绑定则禁止）

### `GET /campuses/mine`
- 说明：当前用户绑定校区（销售选校区页使用）

---

## 2) 角色管理 `roles`

### `GET /roles`
- 说明：角色列表（分页 + 按编码/名称筛选）

### `POST /roles`
- 说明：创建角色（`code` 唯一）
- Body：
```json
{
  "code": "sales",
  "name": "销售"
}
```

### `PATCH /roles/:id`
- 说明：更新角色
- Body（可选字段）：
```json
{
  "code": "sales",
  "name": "销售顾问"
}
```

### `DELETE /roles/:id`
- 说明：删除角色（若有用户引用则禁止）

---

## 3) 用户管理 `users`

### `GET /users`
- 说明：用户列表（分页 + 多条件筛选）
- Query：
  - `page` / `pageSize`
  - `username`（模糊）
  - `roleId`
  - `status`
  - `campusId`（按绑定校区筛选）

### `GET /users/:id`
- 说明：用户详情（含角色和绑定校区）

### `POST /users`
- 说明：创建用户（用户名唯一、密码哈希、支持绑定校区）
- Body：
```json
{
  "username": "admin2",
  "password": "123456",
  "roleId": "uuid",
  "status": "active",
  "campusIds": ["uuid1", "uuid2"]
}
```

### `PATCH /users/:id`
- 说明：更新用户（角色/状态/重置密码/同步 `user_campuses`）
- Body（可选字段）：
```json
{
  "roleId": "uuid",
  "status": "inactive",
  "password": "newPassword",
  "campusIds": ["uuid1"]
}
```

### `DELETE /users/:id`
- 说明：删除用户
- 业务限制：
  - 禁止删除当前登录用户自身
  - 禁止删除“最后一个管理员”

---

## 4) 日志管理 `logs`

### `GET /logs`
- 说明：日志列表（分页 + 筛选）
- Query：
  - `page` / `pageSize`
  - `operatorId`
  - `module`
  - `campusId`
  - `startTime` / `endTime`（ISO 时间）

### `GET /logs/:id`
- 说明：日志详情

### `DELETE /logs/:id`
- 说明：删除日志（软删：写 `deleted_at`）

### `GET /logs/operators`
- 说明：日志操作人下拉数据

---

## 5) 操作日志落库约定

- 模块写操作统一通过 `OperationLogsService.create(...)`
- 当前已接入：
  - 校区：`create` / `update` / `delete`
  - 角色：`create` / `update` / `delete`
  - 用户：`create` / `update` / `delete`

---

## 6) 前端页面对齐

- `/users` 用户管理
- `/roles` 角色管理
- `/campuses` 校区管理
- `/logs` 日志管理（`admin` / `owner`）
