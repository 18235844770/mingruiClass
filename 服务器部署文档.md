# 明睿教务系统服务器部署文档

本文档基于当前仓库结构（`frontend` + `backend`）编写，适用于 **Linux 服务器（推荐 Ubuntu 22.04）** 的生产部署。

## 1. 项目架构说明

- 前端：Vue 3 + Vite，构建产物为静态文件
- 后端：NestJS，默认端口 `3000`，全局 API 前缀 `/api`
- 数据库：PostgreSQL（通过 Prisma 管理）
- 前端请求：统一访问 `/api`，由 Nginx 反向代理到后端

## 2. 服务器准备

建议准备一台最小配置如下的云服务器：

- CPU：2 核
- 内存：4 GB
- 系统盘：40 GB+

### 2.1 服务器环境准备（上线前）

为减少后续部署和运维问题，建议先完成以下环境初始化：

1) 主机与网络

- 绑定公网 IP，配置域名 A 记录到服务器 IP
- 云安全组放行端口：`22`（SSH）、`80`（HTTP）、`443`（HTTPS）
- 若使用数据库云服务，额外放行数据库端口并限制来源 IP

2) 系统初始化

```bash
sudo apt update && sudo apt upgrade -y
sudo timedatectl set-timezone Asia/Shanghai
sudo apt install -y unzip wget net-tools ufw
```

3) 创建业务部署账号（可选但推荐）

```bash
sudo adduser deploy
sudo usermod -aG sudo deploy
```

说明：

- 生产环境建议避免直接使用 `root` 进行日常部署
- 后续应用目录可统一放在 `/opt/mingruiClass`

4) SSH 与防火墙（按需）

```bash
# 开启 UFW，并仅放行必要端口
sudo ufw allow OpenSSH
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable
sudo ufw status
```

5) 基础连通性检查

```bash
ping -c 4 8.8.8.8
curl -I https://www.baidu.com
df -h
free -h
```

以上检查通过后，再继续安装运行时与数据库软件。

安装基础软件（Node.js、Nginx、PostgreSQL、pm2）：

```bash
sudo apt update
sudo apt install -y curl git nginx postgresql postgresql-contrib
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pm2
```

确认版本：

```bash
node -v
npm -v
nginx -v
psql --version
pm2 -v
```

## 3. 获取代码并安装依赖

```bash
cd /opt
```

```bash

```

## 4. 配置 PostgreSQL 数据库

### 4.1 创建数据库与账号

优先使用仓库脚本（推荐）：

```bash
sudo -u postgres psql -f /home/mingrui/backend/scripts/init-db.sql
```

或手工创建：

```bash
sudo -u postgres psql
```

在 `psql` 里执行（请替换密码）：

```sql
CREATE USER mingrui_user WITH PASSWORD 'StrongPassword_ChangeMe';
CREATE DATABASE mingrui_class OWNER mingrui_user;
GRANT ALL PRIVILEGES ON DATABASE mingrui_class TO mingrui_user;
\q
```

### 4.2 配置后端环境变量

```bash
cd /home/mingrui/backend
cp .env.example .env
```

编辑 `backend/.env`：

```env
DATABASE_URL="postgresql://mingrui_user:StrongPassword_ChangeMe@127.0.0.1:5432/mingrui_class?schema=public"
JWT_SECRET="replace-with-a-very-strong-random-secret"
JWT_EXPIRES_IN="1d"
CORS_ORIGINS="https://your-domain.com"
PORT=3000
```

说明：

- `JWT_SECRET` 必须替换为强随机字符串
- `CORS_ORIGINS` 填写你的前端域名，多个域名用英文逗号分隔
- 使用 Nginx 同域名反代时，通常仅保留前端域名即可

## 5. 初始化数据库结构与种子数据


## 7. 构建前端并交给 Nginx 托管

### 7.1 构建前端

```bash
cd /opt/mingruiClass/frontend
npm run build
```

构建完成后，静态资源目录为 `frontend/dist`。

### 7.2 配置 Nginx

创建站点配置：

```bash
sudo nano /etc/nginx/sites-available/mingrui.conf
```

写入以下内容（请替换域名）：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /opt/mingruiClass/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置并重载：

```bash
sudo ln -s /etc/nginx/sites-available/mingrui.conf /etc/nginx/sites-enabled/mingrui.conf
sudo nginx -t
sudo systemctl reload nginx
```

如果默认站点冲突，可删除默认配置：

```bash
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

## 8. HTTPS（推荐）

使用 Certbot（Let’s Encrypt）：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

证书自动续期测试：

```bash
sudo certbot renew --dry-run
```

## 9. 部署后验证清单

### 9.1 后端健康检查

```bash
curl http://127.0.0.1:3000/api/health
```

期望返回：

```json
{"ok":true,"service":"mingrui-backend"}
```

### 9.2 前端访问检查

- 浏览器访问：`https://your-domain.com`
- 页面加载正常且可调用登录接口

### 9.3 API 反代检查

```bash
curl https://your-domain.com/api/health
```

返回结果应与本地后端健康检查一致。


## 11. 常见问题排查

- 前端白屏：检查 `nginx` 的 `root` 是否指向 `frontend/dist`
- API 404：确认后端全局前缀是 `/api`，且 Nginx 反代为 `/api/`
- 登录 401：检查 `JWT_SECRET` 是否在重启后端前后保持一致
- 跨域失败：确认 `CORS_ORIGINS` 包含当前前端域名
- 数据库连接失败：检查 `DATABASE_URL`、PostgreSQL 监听地址和防火墙
