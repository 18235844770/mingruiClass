# 第六阶段：销售模块（学生与课程）— 任务拆分（执行版）

> 依据：`阶段任务清单.md` 第六阶段、`产品设计文档V2.0.md`。  
> **前置**：第四/五阶段已完成（登录、权限、校区上下文、管理员治理模块可用）。

---

## 阶段目标（不变）

完成销售侧**学生与课程录入管理**闭环：学生列表筛选 + 学生创建编辑删除 + 课程多条录入 + 详情聚合展示，并确保校区隔离与创建人规则生效。

---

## 6.1 学生列表（Students List）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 6.1.1 | 列表接口增强 | `GET /api/students` 支持筛选：`name`、`phone`、`paidStatus`、`lowHours`（<3）、`negativeHours`（<0） |
| 6.1.2 | 剩余课时聚合 | 列表返回每个学生课程总剩余（按未删除课程求和） |
| 6.1.3 | 标签状态 | 返回衍生标签：`isLowHours`、`isNegativeHours`、`isArrears`（欠费） |
| 6.1.4 | 前端列表页 | 表格 + 筛选区 + 分页；低课时、欠费、负课时高亮展示 |

---

## 6.2 学生创建（Create Student）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 6.2.1 | DTO 校验 | `name` 必填；`phone` 可空；`totalAmount` 字符串金额；`paidStatus` 布尔；`remark` 可空 |
| 6.2.2 | 同校区重名校验 | 同一 `campus_id` + `name`（未软删）唯一，冲突返回中文可读错误 |
| 6.2.3 | 创建人与校区规则 | 销售创建学生时强制 `campusId = 当前上下文`，`createdBy = 当前销售` |
| 6.2.4 | 前端录入表单 | 新建弹窗/页支持姓名、手机号、总金额、是否交费、备注 |

---

## 6.3 课程录入（Student Courses）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 6.3.1 | 课程新增接口 | `POST /api/students/:id/courses`（或复用 `student-courses` 模块） |
| 6.3.2 | 课程字段 | `courseName`、`coursePrice`、`totalHours`、`remainingHours`、`courseType`、`remark` |
| 6.3.3 | 多条课程录入 | 前端支持一次提交多条课程，后端事务保存 |
| 6.3.4 | 规则边界 | 明确不校验“课程价格总和 == 学生总金额” |
| 6.3.5 | 日志记录 | 课程新增/编辑/删除写 `operation_logs`（module 建议 `student_courses`） |

---

## 6.4 学生编辑与删除

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 6.4.1 | 编辑接口 | `PATCH /api/students/:id`，可改姓名、手机号、总金额、交费状态、备注 |
| 6.4.2 | 删除策略 | 软删 `students.deleted_at`；级联课程按现有策略（建议课程也软删） |
| 6.4.3 | 权限与范围 | 销售仅能编辑/删除自己创建且当前校区可见的学生 |
| 6.4.4 | 前端交互 | 编辑入口、删除二次确认、错误提示透传后端 message |

---

## 6.5 学生详情页

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 6.5.1 | 详情接口 | `GET /api/students/:id` 返回基础信息 + 课程列表 + 消课记录摘要 |
| 6.5.2 | 课程列表展示 | 展示课程价格、总课时、剩余课时、类型、备注 |
| 6.5.3 | 消课历史区域 | 展示历史消课记录（只读，完整功能在第七阶段） |
| 6.5.4 | 前端详情页 | 学生基本信息卡 + 课程表 + 历史记录区 |

---

## 6.6 横切要求（第六阶段）

| 序号 | 任务 | 说明 |
| --- | --- | --- |
| 6.6.1 | 分页返回统一 | 列表接口统一 `{ data, total, page, pageSize }` |
| 6.6.2 | 时间格式统一 | 前端展示统一 `yyyy-mm-dd hh:mm` |
| 6.6.3 | 错误信息可读 | 重名、越权、参数错误均返回可读中文 message |
| 6.6.4 | 日志可追溯 | 学生与课程写操作落 `operation_logs` |
| 6.6.5 | 校区隔离优先 | 全流程遵守当前校区上下文，禁止跨校区写入 |

## 6.7 后端实现建议（Nest）

| 模块 | 任务建议 |
| --- | --- |
| `StudentsModule` | 扩展 list/detail/create/update/remove；补筛选 DTO 与范围校验 |
| `StudentCoursesModule` | 新增批量创建课程接口；必要时补 update/remove |
| `LogsModule` | 复用 `OperationLogsService` 记录学生/课程写操作 |

---

## 6.8 前端页面建议（销售）

| 路由 | 页面 | 备注 |
| --- | --- | --- |
| `/students` | 学生管理 | 列表筛选 + 新建/编辑/删除 + 详情入口 |
| `/students/:id`（可选） | 学生详情 | 课程列表 + 历史消课（只读） |

> UI 建议沿用 Ant Design Vue：`Table` + `Form` + `Modal/Drawer`。

---

## 6.9 验收标准（与总清单对齐）

- [ ] 学生列表支持姓名/手机号/交费状态/低课时/负课时筛选
- [ ] 学生可新增、编辑、删除；姓名同校区唯一生效
- [ ] 课程支持一次录入多条，字段完整可保存
- [ ] 学生详情可查看基础信息、课程和历史消课摘要
- [ ] 销售仅能在当前校区 + 自己数据范围内操作

---

## 6.10 与第七阶段边界

- 第六阶段仅完成“学生与课程管理”主链路。  
- 消课扣减、低课时自动提醒、重复消课提醒等放到第七阶段。

---

## 建议执行顺序

1. 学生列表筛选 + 分页（先把可见性做对）  
2. 学生创建 + 重名校验 + 校区/创建人规则  
3. 学生编辑/删除（含软删）  
4. 课程多条录入接口与前端弹窗  
5. 学生详情聚合（课程 + 消课摘要）  
6. 全链路联调与第 6.9 自检
