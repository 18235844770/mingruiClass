# 3.5 数据库迁移执行与验证清单

在 **`backend/.env`** 中配置可用的 `DATABASE_URL` 后，按顺序执行并勾选。

## 3.5.1 环境

- [ ] PostgreSQL 可连接，`DATABASE_URL` 与 `backend/.env.example` 格式一致  
- [ ] 若库中仍是**旧版仅三表**结构，优先使用空库或 `npx prisma migrate reset`（**清空数据**）后再迁

## 3.5.2 应用迁移

在 `backend` 目录：

```bash
npx prisma migrate deploy
```

或开发环境（可生成客户端并交互命名）：

```bash
npx prisma migrate dev
```

- [ ] 迁移 `20260401120000_phase3_business_tables` 已应用  
- [ ] 迁移 `20260402120000_soft_delete_and_partial_unique` 已应用  

## 3.5.3 生成客户端与种子

```bash
npx prisma generate
npm run prisma:seed
```

- [ ] `prisma generate` 无报错  
- [ ] `prisma:seed` 无报错，控制台输出含 `Seed OK` 与 `demo` 信息  

## 3.5.4 结构核对（任选一种）

- [ ] `npx prisma studio` 中可见：`roles`、`users`、`campuses`、`students`、`student_courses`、`course_consumptions`、`operation_logs` 等表  
- [ ] `students` 表存在 `deleted_at`；存在部分唯一索引 `students_campus_id_name_active_key`（仅 PostgreSQL 元数据可见，Studio 可能不显示定义，可用 SQL 查 `pg_indexes`）  

**可选 SQL（在 psql 中）**：

```sql
\d students
SELECT indexname, indexdef FROM pg_indexes WHERE tablename = 'students';
```

## 3.5.5 与现有骨架衔接说明

- 若曾使用**无 `roles` 表、用户含 `roleCode` 字符串**的旧 schema，**不能**直接 `migrate` 升级，请空库或 `migrate reset` 后重新 `deploy` + `seed`。  
- 本仓库当前后端代码已按 **`users.role_id` + `roles.code`** 实现登录 JWT，与现 schema 一致。
