# 明睿教务系统服务器依赖安装文档

本文档用于在 **Ubuntu 22.04 LTS** 服务器上完成项目运行依赖安装，适用于本仓库的前后端部署（`frontend` + `backend`）。

## 1. 安装目标

需要安装并确认以下依赖：

- Git（拉取代码）
- Node.js 20.x + npm（前后端运行时）
- PM2（后端进程守护）
- Nginx（前端静态托管与反向代理）
- PostgreSQL（业务数据库）

## 2. 基础环境初始化

```bash
sudo apt update && sudo apt upgrade -y
sudo timedatectl set-timezone Asia/Shanghai
sudo apt install -y curl wget ca-certificates gnupg lsb-release software-properties-common
```

## 3. 安装 Git

```bash
sudo apt install -y git
git --version
```

## 4. 安装 Node.js 20.x 与 npm

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v
npm -v
```

说明：

- 建议 Node.js 版本不低于 `20.x`
- 若曾安装旧版 Node.js，请先清理后再安装

## 5. 安装 PM2

```bash
sudo npm install -g pm2
pm2 -v
```

配置 PM2 开机自启（部署完成后执行）：

```bash
pm2 startup
pm2 save
```

## 6. 安装 Nginx

```bash
sudo apt install -y nginx
nginx -v
sudo systemctl enable nginx
sudo systemctl start nginx
sudo systemctl status nginx --no-pager
```

## 7. 安装 PostgreSQL

```bash
sudo apt install -y postgresql postgresql-contrib
psql --version
sudo systemctl enable postgresql
sudo systemctl start postgresql
sudo systemctl status postgresql --no-pager
```

## 8. 可选工具（推荐）

```bash
sudo apt install -y unzip net-tools ufw jq
```

用途说明：

- `unzip`：解压发布包
- `net-tools`：查看端口占用（如 `netstat`）
- `ufw`：主机防火墙管理
- `jq`：处理 JSON 输出

## 9. 一次性安装命令（可直接执行）

```bash
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y curl wget ca-certificates gnupg lsb-release software-properties-common git nginx postgresql postgresql-contrib unzip net-tools ufw jq && \
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && \
sudo apt install -y nodejs && \
sudo npm install -g pm2
```

## 10. 安装完成验收

执行以下命令确认版本和服务状态：

```bash
git --version
node -v
npm -v
pm2 -v
nginx -v
psql --version
sudo systemctl is-active nginx
sudo systemctl is-active postgresql
```

期望结果：

- 所有版本命令均正常返回
- `nginx`、`postgresql` 状态为 `active`

## 11. 常见问题

- Node 安装失败：先执行 `sudo apt update`，再重跑 NodeSource 安装脚本
- npm 全局权限报错：优先使用 `sudo npm install -g pm2`
- Nginx 启动失败：执行 `sudo nginx -t` 检查配置语法
- PostgreSQL 连接失败：确认服务已启动，且本地连接方式正确
