# 完整项目测试流程清单（执行版）

> 适用范围：当前项目全模块（认证、权限、管理员模块、销售模块、提醒、仪表盘、报表、日志）。  
> 测试目标：确保关键业务链路可用、权限与校区隔离正确、统计口径一致、日志可追溯。  
> 执行建议：按本文顺序从上到下执行，失败项记录到缺陷表后回归。

---

## 0. 测试准备

- [ ] 拉取最新代码并安装依赖：`backend`、`frontend`
- [ ] 执行数据库迁移并确认成功
- [ ] 执行种子数据（至少有 `admin/sales/owner` 三角色账号）
- [ ] 启动后端与前端服务，确认可访问
- [ ] 清空浏览器缓存（token、校区选择）后开始测试

---

## 1. 基础可用性检查

- [ ] `GET /api/health` 返回正常
- [ ] 登录页、校区选择页、主布局可正常打开
- [ ] 时间显示格式统一为 `yyyy-mm-dd hh:mm`
- [ ] 错误提示为可读中文（而非泛化错误）

---

## 2. 认证与权限流程

### 2.1 登录与会话
- [ ] `admin` 登录成功并进入管理员默认首页
- [ ] `sales` 登录后强制进入校区选择
- [ ] `owner` 登录成功并进入仪表盘/报表相关入口
- [ ] 无 token 访问受保护页面会跳转登录

### 2.2 角色菜单控制
- [ ] `admin` 仅看到管理员应有菜单
- [ ] `sales` 仅看到销售应有菜单
- [ ] `owner` 仅看到老板应有菜单

### 2.3 接口权限控制
- [ ] `sales` 访问管理员接口被拒绝
- [ ] `owner` 对只读资源可访问、写接口被拒绝（按策略）
- [ ] 越权请求返回 403 且文案合理

---

## 3. 校区隔离与数据范围

- [ ] 销售仅可访问当前校区数据
- [ ] 销售仅可访问自己创建范围数据（学生等）
- [ ] 切换校区后列表数据立即切换
- [ ] 通过手工改 query/header 无法跨校区读取或写入

---

## 4. 管理员模块（阶段5）

### 4.1 用户管理
- [ ] 用户列表分页/筛选可用
- [ ] 新建用户成功（用户名唯一、可绑角色和校区）
- [ ] 编辑用户成功（角色/状态/重置密码/校区）
- [ ] 删除普通用户成功
- [ ] 禁止删除当前登录用户
- [ ] 禁止删除最后一个管理员

### 4.2 角色管理
- [ ] 角色列表分页/筛选可用
- [ ] 新建角色成功，重复 `code` 被拦截
- [ ] 编辑角色成功
- [ ] 被用户引用的角色不可删除

### 4.3 校区管理
- [ ] 校区列表分页/筛选可用
- [ ] 新建/编辑校区成功
- [ ] 被 `students` 或 `user_campuses` 引用的校区不可删除
- [ ] 无引用校区可删除，接口不报 500

### 4.4 日志管理
- [ ] 日志列表分页/筛选（操作人/模块/校区/时间）可用
- [ ] 日志详情可查看
- [ ] 日志删除（软删）可用

---

## 5. 销售模块（学生与课程，阶段6）

### 5.1 学生管理
- [ ] 学生列表支持姓名/手机号/交费状态/低课时/负课时筛选
- [ ] 新增学生成功（手机号可空）
- [ ] 同校区重名校验生效
- [ ] 编辑学生成功
- [ ] 删除学生成功（按软删策略）

### 5.2 课程管理
- [ ] 可为学生新增课程（价格、课时、类型、备注）
- [ ] 支持一次提交多条课程
- [ ] 不校验“课程总价必须等于学生总金额”
- [ ] 学生详情可看到课程列表

---

## 6. 消课管理（阶段7）

### 6.1 消课录入与扣减
- [ ] 选择学生/课程后可提交消课
- [ ] 生成 `course_consumptions` 记录
- [ ] `remaining_hours` 正确扣减
- [ ] 超额消课后允许出现负课时

### 6.2 记录与提醒
- [ ] 消课记录支持按学生/课程/日期筛选
- [ ] 消课时间展示正确可核对
- [ ] 临近时间重复消课仅提示，不阻断提交
- [ ] 消课写操作可在日志中追溯

---

## 7. 低课时提醒与销售首页（阶段8）

- [ ] 销售首页固定展示低课时列表（默认 `<3`）
- [ ] 正确显示低课时/负课时/欠费标签
- [ ] `/low-hours` 页面筛选与分页可用
- [ ] 可跳转学生详情和消课页
- [ ] 提醒数据仍受当前校区 + 创建人范围约束

---

## 8. 仪表盘与报表（阶段9）

### 8.1 仪表盘
- [ ] 指标可展示：在读人数、剩余课时、总课时、总收入、未收入课时费用
- [ ] 支持按校区筛选
- [ ] 支持按时间范围筛选
- [ ] 筛选后数字变化符合预期

### 8.2 报表
- [ ] 报表支持校区/销售/课程类型/日期范围筛选
- [ ] 筛选结果与仪表盘口径一致
- [ ] 导出（Excel/PDF）可用（若已实现）
- [ ] 导出结果与当前筛选一致

---

## 9. 日志与审计（全模块）

- [ ] 用户/角色/校区/学生/课程/消课写操作均写日志
- [ ] 日志 detail 包含关键变更信息（前后值、对象ID、操作人）
- [ ] 日志查询性能可接受（常用筛选）

---

## 10. 稳定性与回归

- [ ] 执行一次完整冒烟回归（登录→管理→销售→统计）
- [ ] 修复缺陷后复测通过
- [ ] 前端构建通过：`npm run build`
- [ ] 后端构建与测试通过：`npm run build`、`npm test`

---
