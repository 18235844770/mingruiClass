# 第四阶段：权限与数据范围说明

## 1. 认证与会话

- 登录接口：`POST /api/auth/login`
  - 成功返回 `accessToken`、`roleCode`
  - JWT 载荷包含 `sub`（用户 ID）、`roleCode`
- 当前用户接口：`GET /api/auth/me`
  - 返回 `id`、`username`、`status`、`roleCode`
- 全局鉴权：`JwtAuthGuard`
  - `@Public()` 放行登录与健康检查

## 2. RequestContext 约定

后端在请求对象上约定以下上下文字段：

- `req.user.userId`：来自 JWT 的当前用户 ID
- `req.user.roleCode`：来自 JWT 的当前角色代码
- `req.campusId`：由 `CampusGuard` 从 `X-Campus-Id` 解析并注入（可选）

说明：

- `sales` 访问业务接口时，`req.campusId` 必须存在且合法。
- `admin` / `owner` 可不传 `X-Campus-Id`；若传则校验格式并注入。

## 3. 校区上下文与数据隔离

统一约定请求头：

- `X-Campus-Id: <uuid>`

`CampusGuard` 行为：

- `sales` 必须携带 `X-Campus-Id`
- `X-Campus-Id` 必须是合法 UUID
- `X-Campus-Id` 必须属于 `user_campuses(user_id, campus_id)` 绑定
- 不满足条件返回 4xx（400/403）

学生数据隔离（已落地）：

- 查询：`GET /api/students`
  - `sales` 自动限定 `campus_id = 当前校区` 且 `created_by = 当前用户`
- 写入：`POST /api/students`
  - `sales` 强制写入 `campus_id = 当前校区`、`created_by = 当前用户`

隔离封装：

- `buildSalesStudentScope(userId, campusId)`

## 4. RBAC 路由策略

角色常量：

- `admin`
- `sales`
- `owner`

全局守卫链：

1. `JwtAuthGuard`
2. `CampusGuard`
3. `RolesGuard`

当前路由权限（第四阶段最小版）：

- `users/*`：`admin`
- `roles/*`：`admin`
- `students/*`：`admin`、`sales`
- `dashboard/*`：`owner`
- `reports/*`：`owner`
- `logs/*`：`owner`

## 5. 前端联动约定

- 登录成功后：
  - `sales` 跳转选校区
  - `admin` 跳转用户管理
  - `owner` 跳转仪表盘
- Axios 自动注入：
  - `Authorization: Bearer <token>`
  - `X-Campus-Id`（有选校区时）
- 路由守卫：
  - 无 token -> 登录页
  - `sales` 无校区 -> 选校区页
  - 角色不匹配 -> 回当前角色默认页
- 错误处理：
  - `401`：清会话并跳登录
  - `403`：跳转当前角色默认页

## 6. 自检步骤（可执行）

前置：

- 已执行数据库迁移与 seed
- 前后端已启动

### 6.1 销售 A / 销售 B 隔离

1. 使用销售 A 登录并选择校区 C1
2. 调 `POST /api/students` 新建学生 S-A
3. 切换销售 B（同校区 C1），调 `GET /api/students`
4. 预期：结果中看不到 S-A

### 6.2 销售切校区隔离

1. 销售 A 在 C1 创建学生 S-C1
2. 销售 A 切到 C2（更换 `X-Campus-Id`）
3. 调 `GET /api/students`
4. 预期：仅返回 C2 且 `created_by = 销售A` 的数据

### 6.3 老板只读范围

1. 老板登录
2. 访问 `GET /api/dashboard/summary`、`GET /api/reports`、`GET /api/logs`
3. 预期：可访问
4. 访问 `GET /api/students`
5. 预期：403

### 6.4 管理员系统配置范围

1. 管理员登录
2. 访问 `GET /api/users`、`GET /api/roles`
3. 预期：可访问
4. 访问 `GET /api/dashboard/summary`
5. 预期：403（当前阶段按 owner 独占）
