# 第五阶段：管理员模块 — 任务拆分（执行版）

> 依据：`开发流程.md` 第五阶段、`阶段任务清单.md`、`产品设计文档V2.0.md`。  
> **前置**：第四阶段已完成——**仅 `admin` 角色**可访问本阶段管理接口；前端管理菜单仅对管理员展示。

---

## 阶段目标（不变）

完成**用户、角色、校区、操作日志**的治理能力：后端 REST + 管理端页面；表格分页筛选；删除二次确认；关键写操作落库 `operation_logs`。

---

## 5.1 用户管理（Users）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 5.1.1 | 列表 | `GET /api/users`：分页（`page`/`pageSize`）、筛选（用户名、角色、状态、可选校区） |
| 5.1.2 | 详情 | `GET /api/users/:id`：含角色、绑定校区列表 |
| 5.1.3 | 创建 | `POST /api/users`：用户名唯一、密码哈希、`roleId`、可选 `status`、**绑定校区 ID 列表** |
| 5.1.4 | 更新 | `PATCH /api/users/:id`：改角色、状态、重置密码（可选）、**同步 user_campuses** |
| 5.1.5 | 删除 | `DELETE /api/users/:id`：禁止删当前登录用户自身或最后一个管理员（策略需定稿） |
| 5.1.6 | 前端页 | 用户列表、新建/编辑弹窗或页、详情、删除二次确认 |
| 5.1.7 | 日志 | 用户增删改写入 `operation_logs`（module=`users`，detail 含变更摘要） |

**数据**：`users`、`user_campuses`、`roles`；密码仅存 `password_hash`。

---

## 5.2 角色管理（Roles）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 5.2.1 | 列表 | `GET /api/roles`：分页；展示 `code`、`name` |
| 5.2.2 | 创建 | `POST /api/roles`：`code` 唯一（建议枚举 `admin`/`sales`/`owner` 或扩展） |
| 5.2.3 | 更新 | `PATCH /api/roles/:id`：名称、**权限配置**（若用 JSON 字段 `permissions` 存 RBAC 点） |
| 5.2.4 | 删除 | `DELETE /api/roles/:id`：若有用户引用则禁止删除或先解绑 |
| 5.2.5 | 前端页 | 角色列表、表单、权限勾选（与产品设计「用户/学生/消课/报表/日志」等模块对齐） |
| 5.2.6 | 日志 | 角色增删改写 `operation_logs`（module=`roles`） |

**说明**：若前期固定三角色且不改表结构，可先做**只读列表 + 权限 JSON 编辑**；完整「增删角色」按产品需要放开。

---

## 5.3 校区管理（Campuses）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 5.3.1 | 列表 | `GET /api/campuses`（管理端全量）：分页、按名称/状态筛选 |
| 5.3.2 | 创建 | `POST /api/campuses`：名称、状态 |
| 5.3.3 | 更新 | `PATCH /api/campuses/:id` |
| 5.3.4 | 删除 | `DELETE /api/campuses/:id`：若存在 `students` / `user_campuses` 引用则 **RESTRICT** 或提示不可删（与库上外键一致） |
| 5.3.5 | 前端页 | 校区列表、表单、删除二次确认 |
| 5.3.6 | 日志 | 校区增删改写 `operation_logs`（module=`campuses`） |

**注意**：`GET /api/campuses/mine`（销售）与「管理端全量校区列表」**路由或权限**需区分，避免销售越权。

---

## 5.4 日志管理（Operation Logs）

| 序号 | 任务 | 说明 / 验收 |
| --- | --- | --- |
| 5.4.1 | 列表 | `GET /api/logs`：分页、筛选（操作人、模块、时间范围、可选 `campus_id`） |
| 5.4.2 | 详情 | `GET /api/logs/:id` |
| 5.4.3 | 删除 | `DELETE /api/logs/:id`：产品为「删除日志」——建议 **软删**（`deleted_at`，第三阶段已预留）或物理删（与 ER 说明一致） |
| 5.4.4 | 前端页 | 日志表格、详情抽屉、删除二次确认 |
| 5.4.5 | 权限 | 管理员、老板可查（若老板只读，与第四阶段策略一致）；删除仅管理员或按产品 |

**写入**：本阶段其他模块的写操作应调用统一的 `LogsService.create(...)`，避免重复代码。

---

## 5.5 横切要求（全模块）

| 序号 | 任务 | 说明 |
| --- | --- | --- |
| 5.5.1 | 分页与筛选 | 所有列表接口统一 query 约定与返回结构（如 `{ data, total, page, pageSize }`） |
| 5.5.2 | 删除二次确认 | 前端 Modal；后端可幂等删除 |
| 5.5.3 | 操作日志 | 用户/角色/校区/日志 的 **POST/PATCH/DELETE** 均写 `operation_logs`；**读操作是否记日志**按产品（默认可不记或仅记敏感读） |
| 5.5.4 | DTO 校验 | `class-validator`；错误信息中文可读 |
| 5.5.5 | 权限守卫 | 上述接口一律 **`@Roles('admin')`**（或权限点细粒度），与第四阶段衔接 |

---

## 5.6 后端模块划分建议

| Nest 模块 | 职责 |
| --- | --- |
| `UsersModule` | UsersController + UsersService |
| `RolesModule` | RolesController + RolesService |
| `CampusesModule` | 扩展管理端方法；或与 `mine` 分 Controller |
| `LogsModule` | LogsController + LogsService（读删） |

共用：`PrismaService`、`LogsService`（或 `OperationLogsService`）。

## 5.7 前端页面清单（管理员）

| 路由 | 页面 |
| --- | --- |
| `/users` | 用户管理 |
| `/roles` | 角色管理 |
| `/campuses` | 校区管理（与「我的校区」区分文案） |
| `/logs` | 日志管理 |

表格：**Ant Design Vue** `Table` + `Form` + 分页器；与现有 `MainLayout` 菜单对齐（仅 `admin` 可见）。

## 5.8 验收标准（与清单一致）

- [x] 管理员可完成：用户/角色/校区的创建、编辑、删除（在业务规则允许范围内）  
- [x] 日志可查询、可看详情、可按产品删除  
- [x] 删除均有二次确认；写操作可追溯至 `operation_logs`  
- [x] 列表均支持分页与主要筛选条件  

---

## 5.9 与第六阶段的边界

- 第五阶段**不包含**学生、课程、消课业务表单的完整实现（属第六阶段）。  
- 第五阶段可为「测试数据」在库中手动或通过 seed 准备用户/校区，供第六阶段联调。

---

## 建议执行顺序

1. 抽取 **`OperationLogsService`** + DTO，保证写日志统一  
2. **校区管理**（表结构已就绪，接口最简单闭环）  
3. **角色管理**（依赖权限模型是否已定）  
4. **用户管理**（依赖角色 + 校区绑定）  
5. **日志查询与删除**  
6. 前端四页联调 + 5.8 自检
