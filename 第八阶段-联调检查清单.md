# 第八阶段联调检查清单（低课时提醒与销售首页）

> 目标：对齐 `第八阶段-任务拆分.md` 的 8.9 验收项，完成低课时提醒链路前后端联调自检。

---

## 0. 联调前准备

- [ ] 后端已启动（`/api` 前缀可访问）
- [ ] 前端已启动并可使用销售账号登录
- [ ] 销售账号已绑定至少一个校区
- [ ] 已有若干学生与课程数据，包含：`remaining_hours < 3` 与 `remaining_hours < 0` 样本

---

## 1. 销售首页提醒区（`/dashboard`）

### 用例 1：固定展示与空态

1. 销售登录后进入首页
2. 检查“低课时提醒”区域

预期：

- [ ] 首页固定展示低课时提醒卡片
- [ ] 有数据时展示列表；无数据时展示空态文案
- [ ] 首页可点击“查看全部”进入 `/low-hours`

### 用例 2：标签与排序

预期：

- [ ] 列表正确标记 `低课时`、`负课时`、`欠费`
- [ ] 排序符合“剩余课时升序”（剩余更少优先）

---

## 2. 低课时独立页（`/low-hours`）

### 用例 3：列表与筛选

1. 进入 `低课时提醒` 页面
2. 分别测试姓名/手机号/交费状态/仅负课时筛选

预期：

- [ ] 默认仅返回低课时学生（`remaining_hours < 3`）
- [ ] 各筛选条件生效且可组合使用
- [ ] 标签口径与学生页一致（低课时/负课时/欠费）

### 用例 4：分页与展示

预期：

- [ ] 分页切换生效
- [ ] 每页条数切换生效
- [ ] 关键展示字段完整（姓名、手机号、交费状态、剩余课时、标签）

---

## 3. 业务跳转联动

### 用例 5：跳转学生详情

1. 在 `/low-hours` 点击“学生详情”
2. 跳转到 `/students` 后观察页面状态
3. 点击“返回低课时页”

预期：

- [ ] 学生页自动打开对应学生详情
- [ ] 页面顶部显示“来自低课时提醒”提示
- [ ] 返回后保留原筛选与分页状态

### 用例 6：跳转消课页

1. 在 `/low-hours` 点击“去消课”
2. 跳转到 `/consumptions` 后观察表单与筛选
3. 点击“返回低课时页”

预期：

- [ ] 消课页自动预选学生（课程可按业务继续选择）
- [ ] 页面顶部显示“来自低课时提醒”提示
- [ ] 返回后保留原筛选与分页状态

---

## 4. 权限与数据范围

### 用例 7：销售范围校验

预期：

- [ ] 提醒数据仅包含当前校区 + 当前销售范围数据
- [ ] 通过 URL 参数无法读取跨校区或他人数据

### 用例 8：角色访问控制

预期：

- [ ] `sales` 可访问 `/dashboard` 与 `/low-hours`
- [ ] `owner` 可访问 `/dashboard`（老板首页占位，不展示销售提醒）
- [ ] `admin` 不暴露销售低课时菜单

---

## 5. 回归检查（与第六/七阶段联动）

### 用例 9：消课后提醒实时性

1. 在 `消课管理` 执行一次消课，使某课程剩余课时跌破阈值
2. 返回首页或低课时页刷新

预期：

- [ ] 新命中低课时学生可被及时检索并展示
- [ ] 负课时状态同步正确

---

## 6. 第八阶段 8.9 验收打勾

- [ ] 销售首页固定显示低课时学生，数据实时可用
- [ ] 低课时列表可筛选、可分页，标签展示准确
- [ ] 可从提醒列表跳转学生详情与消课页
- [ ] 提醒数据严格遵守当前校区 + 当前销售范围
- [ ] 负课时与欠费状态标记正确
