# 测试流程清单

> 依据文档：`开发流程.md`（第十阶段测试指令）、`第十阶段-任务拆分.md`（10.1~10.10）、`第十阶段-联调检查清单.md`。

## 0. 联调前准备
- [ ] 锁定联调基线（分支、数据库、环境变量）
- [ ] 后端构建通过：`cd backend && npm run build`
- [ ] 后端单测通过：`cd backend && npm run test`
- [ ] 前端构建通过：`cd frontend && npm run build`
- [ ] 准备样本数据（正常/边界/异常）

## 1. 字段与契约联调（10.1）
- [ ] DTO/接口字段名、必填、枚举、默认值一致
- [ ] 空值与边界语义一致（`null`/空串/`0`/负值）
- [ ] 时间格式统一（`yyyy-mm-dd hh:mm`）
- [ ] 金额与课时精度一致

## 2. 表单与错误提示检查（10.2）
- [ ] 必填/长度/类型/范围/重复校验可触发
- [ ] 错误文案可读且位置一致
- [ ] 后端 `message` 可透传（含数组 message）
- [ ] 提交流程防重（防重复点击写入）
- [ ] 新增/编辑/删除成功反馈与列表刷新行为一致

## 3. 权限与数据隔离回归（10.3）
- [ ] `admin` 菜单与接口权限正确
- [ ] `sales` 菜单与接口权限正确
- [ ] `owner` 菜单与接口权限正确
- [ ] 销售仅可见当前校区 + 自己创建范围数据
- [ ] 伪造参数/跨校区访问被拒绝
- [ ] 老板只读策略有效

## 4. 核心业务链路回归（10.4）
- [ ] 学生链路：新增 -> 录课 -> 编辑 -> 删除
- [ ] 消课链路：消课登记 -> 课时扣减 -> 负课时/低课时状态正确
- [ ] 提醒链路：低课时列表、详情跳转、消课联动正确
- [ ] 统计链路：仪表盘与报表口径一致

## 5. 日志与审计检查（10.5）
- [ ] 用户/角色/校区/学生/课程/消课写操作均落日志
- [ ] 可按模块、操作人、时间范围检索日志
- [ ] `detail` 包含关键上下文（对象 ID、前后值）
- [ ] 删除策略（软删/物理删）符合约定

## 6. 报表与导出复核（10.6）
- [ ] 报表筛选准确（校区/销售/课程类型/日期范围）
- [ ] Excel 导出与页面筛选结果一致
- [ ] PDF 导出与页面筛选结果一致
- [ ] 样本对账通过（DB 聚合 vs API summary）
- [ ] 常用查询与导出耗时可接受

## 7. 缺陷管理与回归闭环（10.7）
- [ ] 缺陷分级（P0/P1/P2）
- [ ] 每条缺陷完成：复现 -> 修复 -> 自测 -> 回归 -> 关闭
- [ ] 每次修复后执行受影响模块回归
- [ ] 上线前 P0/P1 清零

## 8. 自动化增强（10.8，可选）
- [ ] API smoke：`cd backend && npm run smoke:phase10`
- [ ] 报表样本复核：`cd backend && npm run verify:reports-sample`
- [ ] PDF 样本复核：`cd backend && npm run verify:reports-pdf-sample`
- [ ] 补充单测/E2E/CI 门禁（build/test/lint）

## 9. 必测场景（开发流程强制覆盖）
- [ ] 账号与权限：管理员登录、销售登录并选校区、老板登录、无权限访问拦截
- [ ] 学生管理：同校区重名校验、跨校区同名可创建、手机号为空可创建、多课程添加
- [ ] 消课：正常消课、超额消课、负课时展示、重复消课核查、消课记录查询
- [ ] 业务提醒：低课时提醒、欠费标签、销售首页固定提醒
- [ ] 统计：仪表盘数字正确、报表筛选正确、未收入课时费用计算正确

## 10. 验收门槛（10.10）
- [ ] 关键业务流程全链路可用（学生/课程/消课/提醒/统计）
- [ ] 高优先级缺陷清零（P0/P1）
- [ ] 权限与校区隔离规则全面生效
- [ ] 日志追溯与报表导出能力可用
- [ ] 验收资料完整（测试清单、缺陷修复记录、最终验收报告）
